Title: From Build Wall to Silicon Ceiling: Zero-Registry Plugin Systems and Branchless Behavioral Dispatch in C++

Abstract:
If you have ever triggered a 200-file recompile by changing a single header, or hit a throughput wall when thousands of objects change behavior under load, this session is for you. The Capability Routing Grid (CRG) is an original linker-driven architectural pattern that solves both problems: it eliminates centralized registries so that modules self-register with zero shared headers, and it models behavioral dispatch as a branchless coordinate lookup so that the hardware prefetcher never loses its stream. These techniques generalize across domains: plugin architectures in tooling, behavioral routing in robotics and simulation, rule engines in finance — any system that requires Mutual Anonymity between its core and the capabilities it orchestrates.

CRG is a pay-as-you-go architecture: its pillars are independent — adopt one, some, or all, without coupling between them.

Pillar 1 — Linker-Driven Discovery: Declare a variable in a .cpp file, and it registers itself. No Init() function, no shared header, no central registry. The result is a fully decoupled plugin system obtained as a fundamental property of the architecture — not a framework built on top of it.

Beyond discovery, two optional pillars extend the pattern:

Pillar 2 — Orthogonal Transport: A type-erased wrapper (the ModelShell) breaks the fundamental C++ constraint against virtual template methods, providing a clean OOP interface (Invoke/TryInvoke) while strictly separating concrete state from templated logic — with SBO (stack-allocation) guarantees and full Natvis debuggability.

Pillar 3 — N-Dimensional Dispatch: Behavioral routing is modeled as a coordinate lookup within a CapabilityTensor: by applying Horner's Method to contextual axes (e.g. State, Authority, Environment), complex decisions become a pure branchless arithmetic offset. This "Structural Immunity" keeps data static in memory while capabilities are routed to it — preventing cache-thrashing and ensuring the prefetcher never loses its stream.

Using concrete examples from high-performance simulation and data-oriented design, we demonstrate how CRG sustains over 30 GiB/s memory throughput under high state-transition frequency, where traditional in-memory structural reorganization patterns drop by 30% as behavioral churn increases.

Attendees will learn how to:
- Achieve Mutual Anonymity: implement linker-driven registration using UniversalAnchor, supporting both monolithic and DLL topologies — no shared registries.
- Break the Virtual Deadlock: leverage the ModelShell pattern (inspired by Klaus Iglberger's Type Erasure) to decouple data from logic, with SBO (stack-allocation) guarantees and Natvis debuggability.
- Implement Branchless Dispatch: project complex behaviors in O(1) using a flat CapabilityTensor indexed via Horner's method — eliminating searches, branches, and virtual calls in the hot path.
- Apply Decision/Execution Separation: cache the function pointer resolved by the CapabilityTensor at decision frequency, and invoke it directly at execution frequency — reaching the physical RAM bandwidth ceiling.
- Identify the Break-Even Point: determine when structural immunity outperforms any pattern where changing behavior requires restructuring objects in memory, using an interactive browser simulation capped at 150k entities for demonstration purposes.

Outline:
- 00-05: THE HOOK — Hardware-Software Symmetry. Why peak efficiency requires a physical split between instructions (L1I) and data (L1D). The Build Wall: one bool, 38 minutes, every programmer.
- 05-15: LINKER-DRIVEN DISCOVERY — Mutual Anonymity via NodeList and UniversalAnchor. Features self-register without shared headers. CapabilityBinding expands a TypeList across templatized capabilities. Plugin system as a side effect.
- 15-25: ORTHOGONAL TRANSPORT — Breaking the Virtual Deadlock. The ModelShell: type erasure stripped of all logic, SBO-guaranteed. Identity Decoupling: Data, Logic, Binding as orthogonal dimensions. Invoke/TryInvoke as the cold-path OOP illusion.
- 25-35: N-DIMENSIONAL RESOLUTION — From 0D (pure discovery) to ND (contextual routing). Collapsing sparse hashes into DenseTypeID. Resolving N-dimensional coordinates in O(1) via Horner's method into a flat CapabilityTensor.
- 35-45: SYSTEM SYMBIOSIS — Structural Immunity as a performance guarantee. Decision/Execution separation. ActiveCapability: caching resolved dispatch at decision time for raw static invocation at execution time. Three performance tiers: virtual dispatch -> cached pointer -> prefetcher-aligned direct call — each a measurable step toward the RAM bandwidth ceiling.
- 45-55: BENCHMARKS & ANALYSIS — CRG vs. in-memory structural reorganization at 1%/5%/10% state-transition rates. Live visual stress-test (50k entities). Mathematical break-even point and when not to use CRG.
- 55-60: Q&A — ODR-safety, DLL boundaries (implicit link natively supported; explicit link via lightweight bootstrap), and hot-reload integration.
